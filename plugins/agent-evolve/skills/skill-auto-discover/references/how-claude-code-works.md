# Claude Code 如何工作

> 来源：<https://code.claude.com/docs/zh-CN/how-claude-code-works>

## 代理循环

Claude Code 的核心是三阶段循环：**收集上下文 → 采取行动 → 验证结果**。

- 循环根据任务调整：问题可能只需收集，修复需要多次循环，重构需要广泛验证
- 用户可随时中断引导，Claude 自主工作但对输入保持响应

## 模型

| 模型 | 特点 |
|------|------|
| Sonnet | 大多数编码任务 |
| Opus | 复杂架构决策（更强推理） |
| Haiku | 轻量/快速任务 |

切换：`/model`（会话中）或 `claude --model <name>`（启动时）

## 内置工具五类

| 类别 | 能力 |
|------|------|
| 文件操作 | 读取、编辑、创建、重命名 |
| 搜索 | 按模式/正则搜索、代码库探索 |
| 执行 | Shell 命令、git、包管理器 |
| 网络 | 搜索网络、获取文档 |
| 代码智能 | 类型错误、跳转定义、引用（需插件） |

## Claude 可访问的内容

- 项目文件（当前目录及子目录）
- 终端命令（你能运行的它都能）
- Git 状态（分支、未提交更改、提交历史）
- CLAUDE.md（项目特定说明）
- Auto memory（自动保存的模式和偏好）
- 配置的扩展（MCP、skills、subagents、Chrome）

## 执行环境

| 环境 | 代码运行位置 | 用例 |
|------|------------|------|
| 本地 | 你的机器 | 默认，完全访问 |
| 云 | Anthropic VM | 卸载任务、无本地仓库 |
| 远程控制 | 你的机器，浏览器控制 | Web UI + 本地执行 |

## 会话管理

- **会话独立**：每个新会话无历史，靠 auto memory 和 CLAUDE.md 保持学习
- **跨分支**：切换分支后文件内容变，对话历史不变
- **恢复**：`claude --continue` 追加到现有对话
- **分叉**：`claude --continue --fork-session` 新会话 + 到此点的历史
- **检查点**：每次编辑前自动快照，按两次 Esc 回退

## 上下文管理

- 接近限制时自动压缩：先清旧工具输出，再总结对话
- 用 `/compact` 手动触发，可加 focus 参数
- 用 `/context` 查看分布
- Skills 按需加载；subagent 独立上下文

## 权限模式（Shift+Tab 切换）

1. **默认**：编辑和命令前询问
2. **自动接受编辑**：编辑不问，命令仍问
3. **Plan Mode**：仅只读，创建可审批的计划
4. **自动模式**：后台安全检查（研究预览）

## 使用技巧

1. **具体 > 模糊**：引用文件、提及约束、指出模式
2. **提供验证手段**：测试用例、截图、预期输出
3. **探索 > 直接实现**：用 plan mode 先分析，再编码
4. **委派 > 指示**：给上下文和方向，让 Claude 决定细节
5. **对话式迭代**：第一次不对时细化，不用重新开始
